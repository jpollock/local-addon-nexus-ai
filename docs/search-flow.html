2. AI integration — classifyIntentWithAI()

Reads settings.aiProvider and calls the configured provider with a tiny classification prompt. No tool calls, no streaming — just a one-word response.

Provider selection

ProviderDefault modelRequires key?
Ollama LOCALllama3.2No — local
Anthropicclaude-haiku-4-5Yes
OpenAIgpt-4o-miniYes
Googlegemini-1.5-flashYes
None configured—Falls back to regex

Classification prompt (sent to AI)

Classify this search query with exactly one word:
content, metadata, or both.

- content: searching for posts, pages, articles,
  or written content on a topic
- metadata: searching for sites by plugin, theme,
  PHP version, WordPress version, or site config
- both: query spans content AND site config

Query: "what are my oldest php sites"
Answer:
AI responds with one word: metadata. 5 second timeout. Falls back to regex on failure.

Fallback chain

classifyIntentWithAI("what are my oldest php sites")
  → reads settings.aiProvider  = 'ollama'
  → no API key needed
  → calls Ollama /api/chat with classification prompt
  → response: "metadata"  ✓

classifyIntentWithAI("customer onboarding")
  → Ollama responds: "content"  ✓

// Provider not configured:
classifyIntentWithAI("form builder sites")
  → aiProvider = 'anthropic', no key → returns null
  → falls back to classifyIntent() regex heuristic
  → regex: no pattern matches "form builder" → "content"  ⚠ wrong without AI

// But detectMetadataQueryIntent catches it:
detectMetadataQueryIntent("form builder sites")
  → /\bform\s*build/ matches → 'plugin-category'  ✓

3. detectMetadataQueryIntent() — SQL branch selection

Runs after intent classification, before SQL. Inspects the query string and chooses which SQL query to run.

php-sort / wp-sort

Detected by regex patterns. Runs ORDER BY version ASC/DESC.

Pattern detectedSQL run
"oldest php sites"ORDER BY php_version ASC
"old version of PHP"ORDER BY php_version ASC
"outdated PHP"ORDER BY php_version ASC
"sites needing PHP upgrade"ORDER BY php_version ASC
"PHP 7 sites"ORDER BY php_version ASC
"latest PHP"ORDER BY php_version DESC
"old WordPress"ORDER BY wp_version ASC
"latest WP"ORDER BY wp_version DESC

plugin-category

Detected by concept keywords. Runs slug IN (...) against known plugin lists.

Query conceptSearches slugs for
"form builder"contact-form-7, wpforms, gravity-forms, ninja-forms…
"page builder"elementor, beaver-builder, divi, js_composer…
"SEO"wordpress-seo, rank-math, all-in-one-seo-pack…
"e-commerce"woocommerce, easy-digital-downloads…
"caching"litespeed-cache, w3-total-cache, wp-super-cache…
"security"wordfence, sucuri-scanner, bulletproof-security…
"backup"updraftplus, backwpup, duplicator…
"performance"autoptimize, smush, imagify, jetpack-boost…
"analytics"google-analytics-for-wordpress, wp-statistics…

substring (existing)

Falls through when no pattern matches. Runs LIKE '%query%'.

QuerySQL
"elementor"name LIKE '%elementor%'
"woocommerce"name LIKE '%woocommerce%'
"PHP 7.4"php_version LIKE '7.4%'
"WP 6.9"wp_version LIKE '6.9%'
"Astra"name LIKE '%astra%' (themes)
Still limited: "form builder" with no AI → no regex catch → LIKE → no match. With AI: AI routes to 'metadata', category pattern catches "form builder" → IN query works.

6. Sidebar search — separate implementation

The sidebar search (slide-in panel from the Local sites list) is a completely separate implementation with its own AI integration path. It uses the same provider infrastructure but a different — and richer — system prompt.

How it works

User types: "sites using ACF with PHP 7.4"
  → SITE_FINDER_AI_PARSE IPC
  → reads settings.aiProvider + apiKey
  → calls provider with full system prompt
  → AI returns structured JSON:
    {
      "plugins": ["advanced-custom-fields"],
      "phpVersions": ["7.4"]
    }
  → Runs structured filter against fleet
  → Shows matching sites in sidebar

What the AI system prompt understands

FieldAI maps to
pluginsPlugin slug array — "ACF" → "advanced-custom-fields"
themesTheme slug array
phpVersionsVersion strings — "old PHP" → ["7.4", "7.3", "5.6"]
wpVersionsWP version strings
contentQuery"sites about X" → semantic search on indexed content
searchTextExact text in site name or domain
This is more powerful than Search tab for fleet filtering — it returns a structured filter that drives the site list, not just result rows.

7. Configuration — how to enable AI

Option A — Ollama (free, local)

  1. Install Ollama from ollama.ai
  2. Run ollama pull llama3.2
  3. Open Nexus Preferences
  4. Set AI Provider → Ollama
  5. Reload Local addon
No API key needed. Uses http://localhost:11434. Classification adds ~200-500ms per query.

Option B — Anthropic / OpenAI / Google

  1. Get API key from provider
  2. Open Nexus Preferences
  3. Set AI Provider → Anthropic (or other)
  4. Enter API key
  5. Reload Local addon
Uses claude-haiku / gpt-4o-mini / gemini-flash — fast and cheap for classification. Costs fraction of a cent per search.

Option C — No provider (heuristic only)

All 12 regex patterns + detectMetadataQueryIntent() patterns still run. Most common queries work without AI:

  • Exact plugin/version names
  • PHP/WP version sorting ("oldest php")
  • Category concepts ("form builder", "seo")
  • Indexed content (semantic search)
Missing: ambiguous natural language, novel phrasings, complex queries.
