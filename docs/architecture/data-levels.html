⚠ Shared Reactive Store gap: Currently every UI panel manages its own IPC fetch cycle with independent 30-second poll timers (FleetCompletenessWidget, SystemTab, NexusOverview). This means panels can show inconsistent data — e.g. Data Completeness shows 113 Configured while Site Status shows 0 WP version, because they polled at different moments relative to a sync completing. The "should be" column consistently calls for a MobX observable store (or equivalent) in the renderer, fed by main-process push events, so all panels always reflect the same authoritative state.
