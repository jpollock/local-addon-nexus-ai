# MU Plugin Lifecycle

The `nexus-ai-connector-config.php` MU plugin is auto-generated by the addon and installed into `wp-content/mu-plugins/` on each managed WordPress site.

## When it is written

1. **Site start** (`siteStarted` lifecycle hook in `lifecycle-hooks.ts`) — calls `installNexusAiConnectorPlugin()` which writes/updates the MU plugin via filesystem
2. **`setup-ai` MCP tool** — also writes the MU plugin as part of the full AI setup flow
3. **`UPDATE_NEXUS_GATEWAY` IPC** — rewrites when gateway configuration changes

## What it contains

Generated by `src/main/ai-gateway/mu-plugin-template.ts`. Key sections:

- **Webhook constants** (`NEXUS_AI_WEBHOOK_URL`, `NEXUS_AI_AUTH_TOKEN`, `NEXUS_AI_SITE_ID`)
- **Gateway constants** (`NEXUS_AI_GATEWAY_URL`, `NEXUS_AI_GATEWAY_TOKEN`, `NEXUS_AI_PROVIDER`)
- **Caller detection** — backtrace-based plugin/theme/feature identification for AI request tracking
- **Caller header injection** — `http_request_args` filter adds `X-WP-Caller-*` headers to gateway requests
- **Connector approval** — `option_wpai_connector_approvals` filter pre-approves known callers
- **Connector unblock** — `pre_http_request` priority-6 filter unblocks blocked gateway requests
- **Image model preference** — `wpai_preferred_image_models` filter injects local-gateway image models
- **Smart Search backend** — optional `option_wpe_content_engine_option_name` filter (Atlas Search sites only)

## What it does NOT contain

- Real API keys (Anthropic/OpenAI/Google)
- `WP_DEBUG` or `WP_DEBUG_LOG` (removed May 2026 — was causing unbounded debug.log growth)

## Upgrade path

The MU plugin is overwritten on every site start. Adding new functionality: update `mu-plugin-template.ts`, then either wait for the next site start or re-run `setup-ai` to propagate immediately.

For currently-running sites that need an emergency patch, copy the file directly to `wp-content/mu-plugins/nexus-ai-connector-config.php`.
