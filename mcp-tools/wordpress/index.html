    WordPress Tools¶

    WP-CLI integration tools for managing WordPress plugins, themes, core, and more.

    Overview¶

    These tools wrap WP-CLI commands and work on both local and remote (WP Engine) sites via SSH.

    12 tools available:

    • Plugin management (list, activate, deactivate, update, install)
    • Core version check
    • Theme management
    • User management
    • Site health checks
    • Options management
    • Database operations

    wp_plugin_list¶

    List all installed WordPress plugins on a site.

    Access
    LOCAL REMOTE
    Safety Tier
    1 - SAFE
    Returns
    Array of plugin objects with name, status, version

    Input Schema¶

    {
  // Target (provide ONE of these)
  site_id?: string;        // Local site ID
  install_name?: string;   // WPE install name

  // Optional filters
  status?: "all" | "active" | "inactive";  // Default: "all"
}

    Examples¶

    {
  "tool": "wp_plugin_list",
  "arguments": {
    "site_id": "abc123",
    "status": "active"
  }
}

    {
  "tool": "wp_plugin_list",
  "arguments": {
    "install_name": "mysite-prod"
  }
}

    {
  "tool": "wp_plugin_list",
  "arguments": {
    "site_id": "abc123"
  }
}

    Response¶

    {
  "content": [{
    "type": "text",
    "text": "✓ Found 12 plugins (8 active, 4 inactive):\n\nActive Plugins:\n  1. Akismet Anti-Spam 5.3.1\n  2. Classic Editor 1.6.2\n  3. Contact Form 7 5.8\n  ...\n\nInactive Plugins:\n  9. Hello Dolly 1.7.2\n  10. Jetpack 12.9\n  ..."
  }]
}

    Related Tools¶

    • wp_plugin_activate - Activate a plugin
    • wp_plugin_deactivate - Deactivate a plugin
    • wp_plugin_update - Update plugins

    wp_plugin_activate¶

    Activate a WordPress plugin.

    Access
    LOCAL REMOTE
    Safety Tier
    2 - CAUTION
    Returns
    Confirmation message

    Input Schema¶

    {
  // Target (provide ONE of these)
  site_id?: string;        // Local site ID
  install_name?: string;   // WPE install name

  // Required
  slug: string;            // Plugin slug (e.g., "akismet")
}

    Examples¶

    {
  "tool": "wp_plugin_activate",
  "arguments": {
    "site_id": "abc123",
    "slug": "akismet"
  }
}

    {
  "tool": "wp_plugin_activate",
  "arguments": {
    "install_name": "mysite-prod",
    "slug": "woocommerce"
  }
}

    Response¶

    {
  "content": [{
    "type": "text",
    "text": "✓ Activated plugin: Akismet Anti-Spam"
  }]
}

    Error Handling¶

    // Plugin already active
{
  "content": [{
    "type": "text",
    "text": "Warning: Plugin already active: akismet"
  }],
  "isError": false  // Not an error, just a warning
}

// Plugin not found
{
  "content": [{
    "type": "text",
    "text": "Error: Plugin not found: nonexistent\n\nRun wp_plugin_list to see available plugins."
  }],
  "isError": true
}

    clean_database_items¶

    Remove identified database bloat. Defaults to dry-run mode — no changes are made unless dry_run is explicitly set to false.

    Access
    LOCAL
    Safety Tier
    3 - DESTRUCTIVE
    Returns
    Rows affected (dry run) or rows deleted (live run) per category

    Parameters:

    Parameter Type Required Description
    site_id string Yes Local site ID
    items string[] Yes Categories to clean (e.g. ["revisions", "transients", "sessions"])

    Examples¶

  "tool": "clean_database_items",
  "arguments": {
    "site_id": "abc123",
    "items": ["revisions", "transients"],
    "dry_run": true
  }
}

    {
  "tool": "clean_database_items",
  "arguments": {
    "site_id": "abc123",
    "items": ["revisions", "transients", "woo_sessions"],
    "dry_run": false
  }
}

    Response¶

    {
  "content": [{
    "type": "text",
    "text": "✓ Cleanup complete — mysite (DRY RUN)\n\n  revisions:   1,204 rows would be deleted (~18 MB)\n  transients:    892 rows would be deleted (~4 MB)\n\nRun with dry_run=false to apply."
  }]
}

    Usage Patterns¶

    Plugin Workflow¶

    // 1. List plugins to see status
await callTool("wp_plugin_list", {site_id: "abc123"});

// 2. Check for updates (dry run)
await callTool("wp_plugin_update", {
  site_id: "abc123",
  all: true,
  dry_run: true
});

// 3. Update specific plugin
await callTool("wp_plugin_update", {
  site_id: "abc123",
  slug: "akismet"
});

// 4. Verify update
await callTool("wp_plugin_list", {
  site_id: "abc123",
  status: "active"
});

    Site Audit Workflow¶

    // 1. Get WordPress version
await callTool("wp_core_version", {site_id: "abc123"});

// 2. List plugins
await callTool("wp_plugin_list", {site_id: "abc123"});

// 3. Run health check
await callTool("wp_site_health", {site_id: "abc123"});

// 4. Check users
await callTool("wp_user_list", {site_id: "abc123"});

    Local to Remote Workflow¶

    // 1. Export local database
await callTool("wp_db_export", {site_id: "local-abc"});

// 2. Search/replace URLs (dry run)
await callTool("wp_search_replace", {
  site_id: "local-abc",
  search: "localhost",
  replace: "mysite.com",
  dry_run: true
});

// 3. Apply changes
await callTool("wp_search_replace", {
  site_id: "local-abc",
  search: "localhost",
  replace: "mysite.com",
  dry_run: false
});

// 4. Push to WPE
await callTool("local_wpe_push", {
  site: "local-abc",
  remote_install_id: "wpe-prod",
  include_database: true
});

    Error Reference¶

    Error Cause Solution
    Site not found Invalid site_id or install_name Run nexus_list_sites first
    Plugin not found Invalid plugin slug Run wp_plugin_list to see slugs
    WP-CLI not available Site not running Start site with local_start_site
